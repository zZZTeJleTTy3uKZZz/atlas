## Секция 1: ФАЙЛЫ (инженерия контекста, неделя 1)

Основа — 5-уровневый каскад контекста (5-scope cascade) [3]. Правило: **чем глубже файл, тем выше приоритет**. Все < 200 строк (иначе "Lost in the Middle") [3, 4].

### 1.1 Global Scope — `~/.claude/CLAUDE.md`

Шаблон (WHAT/WHY/HOW) [6]:
1. Роль: Оркестратор-одиночка, управляющий портфелем B2B SaaS и внедрений.
2. Стек по умолчанию: Windows 11, Python, JS.
3. Формат мышления: сначала модель данных, потом логика, UI в конце [7].
4. Compound Engineering: никогда не извиняйся, давай только код.
5. Изоляция: паттерн "Fresh Chat Per Task" [2].

Источники: [3, 8, 9].

### 1.2 Project Scope (бизнес-продукты) — `~/Cifro.pro/AGENTS.md`

Шаблон:
1. WHAT: архитектура NP-001..005.
2. Стек: точные версии фреймворков [10, 11].
3. Executable Commands: скрипты для сборки, линтинга, деплоя с полными флагами [11, 12].
4. Паттерны: примеры адаптеров для работы с API.
5. Границы (Don't Touch): файлы конфигурации ядра, которые ИИ запрещено менять без явного приказа [11, 13].

### 1.3 Project Scope (клиенты) — `~/B24_Clients/AGENTS.md`

Шаблон:
1. Контекст: Bitrix24 REST API (лимиты, вебхуки).
2. Правила: запрет на хардкод клиентских ID.
3. SSOT: ссылка на `notion-task-cli` как единственный источник правды по задачам [16].
4. QA: обязательная проверка обработчиков ошибок для интеграций.

### 1.4 Local Secret Scope — `~/Tests/CLAUDE.local.md` (в `.gitignore`!)

Содержимое: временные токены, WIP-заметки, чувствительные пути к локальным серверам [3, 17, 18].

### 1.5 Folder / Imports Scope — `~/Cifro.pro/NP-001/src/CLAUDE.md`

Специфика конкретного модуля или `@imports` для подгрузки глобальных правил (`@docs/security.md`) [3, 17].

## Секция 2: ИНСТРУМЕНТЫ (ROI-оптимизированный стек)

Разделение **"Control Plane"** (управление намерениями) vs **"Storage Surface"** (хранение) [19].

1. **Claude Code — оставить как есть.** ADE уже у вас. Warp 2.0 / Intent не нужны. Multi-agent — через параллельные вкладки терминала в разных git-worktrees. `/compact` (50% контекста) и `/clear` при смене задачи [20, 21]. **ROI: максимальный (настроено).**
2. **Notion + notion-task-cli — оставить.** Linear избыточен, миграция съест энергию активации [22-24].
3. **NotebookLM + Notion AI — интегрировать плотнее.** NotebookLM = аналитический движок, Notion AI = структурирование в экшен-планы [25-27]. **ROI: очень высокий.**
4. **Motion ($34/мес) или Amie (free) — ВНЕДРИТЬ.** Защита календаря при 10+ внедрениях. ИИ-планировщик создаёт блоки для deep work (Control Plane для времени) [24, 28, 29]. **ROI: высокий (защита от СДВГ-паралича).**
5. **Latenode — ВНЕДРИТЬ.** Для Bitrix24 ↔ Notion sync. В отличие от Zapier — плата за CPU time, не за каждую операцию. Встроенные AI-агенты, JS в нодах [30-32]. **ROI: высокий.**

## Секция 4: STANDALONE CAPABILITY GROUP у одиночки

Вы — Оркестратор, но работаете как команда из 4-5 ролей [40].

### 4.1 Agent Personas (через файловое ролевое кэширование) [41]

- **Scribe/PM:** анализирует логи встреч из B24, создаёт `PRD.md` [42].
- **Coder:** запускается в `Cifro.pro/`, читает `AGENTS.md` + `PRD.md`, пишет код [42].
- **QA (Критик):** отдельное окно. Команда: "Найди уязвимости и архитектурные ошибки в PR. Докажи, что он сломает систему" (Adversarial Review) [43].
- **Researcher:** связка NotebookLM + Notion AI.

### 4.2 Fresh Chat Per Task [2, 44]

Никогда не держать один долгий диалог. Протокол:
1. Агент сгенерировал логику → сохранил в `feature.md`.
2. Открыли новую вкладку → подгрузили `feature.md`.
3. Новый агент пишет тесты.

Markdown на диске = API между агентами [2, 45]. 100% защита от Context Rot [44].

## Секция 5: API DRIFT GOVERNANCE

При 10 клиентах любое изменение в API → Schema Drift [46, 47].

**Минимальный setup (Contract-First Development) [47]:**

1. **SSOT:** описать API-взаимодействия с B24 в `openapi.yaml`.
2. **Spectral (линтер):** `spectral lint openapi.yaml` в pre-commit hook.
3. **oasdiff (семантический диффер):** перед релизом `oasdiff changelog base.yaml revised.yaml`.

Optic для нас избыточен — oasdiff закроет 95% рисков [47].

## Секция 6: АНТИ-ПАТТЕРНЫ (что НЕ делать первые 14 дней)

Топ-7 смертельных ошибок соло-оркестраторов [48-50]:

1. **Мега-система (СДВГ-ловушка)** [23, 33]. Не связывать B24+Notion+Claude+Git в монолит. Начать с простых `state.md`.
2. **Прыжок в код без модели данных (AI Autopilot)** [51-53]. Через `CLAUDE.md` → сначала типы/интерфейсы, потом логика [7, 54].
3. **Агентизация детерминированных задач** [42]. Если решается 20-строчным скриптом или вебхуком — не поручать ИИ. LLM нужны только где есть неопределённость [42, 55].
4. **Раздувание CLAUDE.md** [3, 4]. > 200 строк → Claude "забывает" правила. Удалять то, что модель и так знает [4, 56].
5. **Context Collapse** [2]. Длинный тред → галлюцинации. Симптом: странные импорты. Лечение: `/clear` [20].
6. **Denial of Wallet** [38, 43, 50]. Зацикливание агента. Решение: жёсткие лимиты итераций + мониторинг биллинга раз в 3 дня.
7. **Prototype Illusion** [48]. Код из локального чата упадёт на реальных данных B24. Паттерн "Human-on-the-loop" — аппрув перед деплоем [43, 57].
